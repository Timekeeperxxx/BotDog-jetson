# BotDog 交付验收清单

> **版本**：v1.0  
> **用途**：阶段 4 交付收口检查  
> **适用范围**：仅限 `/home/jetson/Project/BOTDOG/BotDog` 仓库内  

---

## 1. 验收目标

确认当前系统已经达到可交付、可维护、可回滚、可审计的产品状态。

验收重点不是“有没有更多功能”，而是：

- 关键链路是否稳定
- 契约是否保持不变
- 失败是否可诊断
- 回滚是否有锚点
- 文档与实现是否一致

---

## 2. 验收前提

以下内容在验收期间视为固定约束：

1. 不修改仓库外文件
2. 不修改 `data/` 中现有 JSON 格式
3. 不修改 ROS2 话题名称、方向和读写语义
4. 不改变已确认正确的业务逻辑
5. 不引入破坏性 API 变更

---

## 3. 必须通过的检查项

### 3.1 启动与系统状态

- 后端可以正常启动
- `/api/v1/system/health` 可访问
- `/api/v1/system/status` 可访问
- `/api/v1/system/startup` 可访问
- 启动摘要与启动快照内容一致

### 3.2 鉴权与错误响应

- 登录流程可用
- 未授权访问会返回统一错误格式
- 错误响应保留原始 `detail`
- 错误响应包含机器可读 `error` 字段

### 3.3 控制与安全

- 控制命令链路可用
- 紧急停止可用
- 控制优先级和 watchdog 行为保持一致
- 安全拒绝场景能返回明确错误

### 3.4 导航、建图与任务

- 导航功能可用
- 建图功能可用
- 任务执行可用
- 建图与导航的互斥行为保持既有约定
- 前端发起导航与任务时，后端能够正确触达对应 ROS2 读写链路

### 3.5 前端交互

- 导航页可用
- 任务编辑与执行可用
- 危险操作有明确确认
- 加载态、空态、错误态表现一致

### 3.6 测试门禁

- 全量测试通过
- 核心回归测试稳定
- 新增测试不依赖脆弱的环境假设

---

## 4. 非必须项

以下内容不是本阶段验收失败条件，但建议继续观察：

- 配置项进一步分域整理
- 前端更细粒度状态管理拆分
- 运维脚本进一步标准化
- 文档的历史残留表述继续清理

---

## 5. 回滚锚点

如果后续改动出现回归，优先使用最近稳定锚点回退。

建议保留的锚点包括：

- 阶段 2 结构收敛锚点
- 阶段 3 测试与运维收口锚点

---

## 6. 验收输出物

验收完成后，应至少保留以下结果：

1. 当前代码状态说明
2. 关键测试结果
3. 仍然存在的低风险技术债列表
4. 后续维护建议
