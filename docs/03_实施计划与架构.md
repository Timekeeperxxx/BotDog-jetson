### 1.2 软件架构

```
浏览器（React + WebRTC WHEP）
    │
    ├── HTTP/WS ──→ FastAPI 后端 (:8000)
    │                 ├── UnitreeB2Adapter → Unitree SDK → B2
    │                 ├── AutoTrackService（YOLOv8 检测 + 决策）
    │                 ├── AIWorker（RTSP 拉流推理）
    │                 ├── ControlArbiter（控制权仲裁）
    │                 └── SQLite（遥测/快照/配置）
    │
    └── WHEP ──→ MediaMTX (:8889)
                    ↑
                FFmpeg（H.265→H.264，15fps，软解转码）
                    ↑
                IP 摄像头 RTSP
```

## 三、核心模块说明

### 3.1 控制仲裁器（ControlArbiter）

负责在多个控制源之间分配优先级，确保安全：

```
优先级：E-STOP > 遥控器(FT24) > Web 手动 > AutoTrack 自动跟踪
```

### 3.2 自动跟踪服务（AutoTrackService）

- **输入**：AIWorker 检测到的目标边界框
- **状态机**：`IDLE → TRACKING → SEARCHING → LOST`
- **输出**：通过 ControlArbiter 发送运动控制指令给 UnitreeB2Adapter

### 3.3 视频流水线

```
IP 摄像头 (H.265 RTSP)
    ↓  FFmpeg
       - -fflags nobuffer（零缓冲）
       - libx264 ultrafast（纯软压）
       - -bf 0（无 B 帧等待）
       - 15fps / 800kbps
       - RTSP 推流
    ↓  MediaMTX
       - RTSP 接收
       - WebRTC WHEP 分发
    ↓  浏览器 <video>（WebRTC）
```

### 3.4 前端键盘控制

全局 `window` 键盘监听，支持：

| 按键 | 命令 | 按键 | 命令 |
|------|------|------|------|
| W | forward | S | backward |
| A | strafe_left | D | strafe_right |
| Q | left（左转） | E | right（右转） |
| Shift | stand（起立） | Ctrl | sit（下蹲） |

松开按键时自动发送 stop 指令，防止机器狗持续运动。

## 四、后续优化方向（可选）

1. **视频延迟进一步降低**：替换摄像头为原生 H.264 输出，MediaMTX 直接代理无需转码，延迟可降至 50ms 以内
2. **硬件加速编码**：为 OrangePi RK3588 适配 `h264_rkmpp` 硬件编码器，降低 CPU 占用
3. **多目标跟踪**：当前 AutoTrackService 只跟踪单一主目标，可扩展为多目标优先级队列
4. **WebRTC 抗抖动调优**：优化 MediaMTX WHEP 的 Jitter Buffer 参数降低浏览器侧延迟
5. **分布式多机协同**：引入多机 task 管理、多通道控制
