BotDog 前端控制实现计划
本实施计划为前端网页控制机器狗功能的开发分阶段实施方案，旨在逐步实现从最简单的控制命令功能到完整的控制平台。

阶段 1：打通最小控制链路
- 前端新增控制面板，支持前后左右、下蹲起立控制按钮。
- 后端新增 HTTP 接口，前后左右、下蹲起立通过 HTTP 发送控制命令。

阶段 2：补齐控制安全底座
- 增加前端控制安全机制，按下持续发命令，松手停止。
- 后端增加 watchdog，超时未收到命令自动 stop。

阶段 3：把控制逻辑从接口调用升级成控制域
- 新增 Control Service 层，统一管理控制请求。
- 增加控制器适配器，统一接口与设备的交互。

阶段 4：为后续迁移和扩展留接口
- 定义设备能力描述，未来可以替换不同设备适配器。
- 为 FT24 / 自动任务等控制源预留扩展接口。
