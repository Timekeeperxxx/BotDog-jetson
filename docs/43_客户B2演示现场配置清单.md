## 三、需要配置的参数（按顺序）

### Step 1：OrangePi 网卡 IP 配置

客户 B2 的网段是 `192.168.123.x`，需要给 OrangePi 的有线网卡配 B2 网段的 IP：

```bash
# 先确认网卡名（你的板子是 enP3p49s0）
ip link show

# 配置 B2 通信 IP
sudo ip addr replace 192.168.123.222/24 dev enP3p49s0

# 添加 DDS multicast 路由（Unitree SDK 必须！）
sudo ip route replace 224.0.0.0/4 dev enP3p49s0
```

验证：

```bash
ping 192.168.123.161   # B2 → 必须通
```

> **⚠️ 注意：如果客户 B2 的 IP 不是 `192.168.123.161`**，需要现场确认！问客户或者用 `nmap -sn 192.168.123.0/24` 扫描。

---

### Step 2：`backend/.env` 关键参数

这是**最核心的配置文件**，当前 `.env` 大部分已经是真机配置。到现场需要**确认/可能调整**的参数：

```ini
# ==================== 关键：必须检查 ====================

# 控制适配器 → 必须是 unitree_b2
CONTROL_ADAPTER_TYPE=unitree_b2

# 网卡名 → 你的开发板是 enP3p49s0（已配好）
UNITREE_NETWORK_IFACE=enP3p49s0

# 数据源 → 关闭模拟
MAVLINK_SOURCE=simulation
SIMULATION_WORKER_ENABLED=false

# AI → 开启
AI_ENABLED=true
AI_RTSP_URL=rtsp://127.0.0.1:8554/cam
AI_MODEL_PATH=models/yolov8n.pt
AI_DEVICE=cpu

# ==================== 演示安全：建议保守速度 ====================

# 前进速度（m/s），首次在客户机上建议先 0.2
UNITREE_B2_VX=0.2

# 转向速度（rad/s），建议先 0.3
UNITREE_B2_VYAW=0.3

# ==================== Watchdog 超时 ====================
# B2 SDK RPC 延迟较大，1500ms 是安全值
CONTROL_WATCHDOG_TIMEOUT_MS=1500
```

> **⚠️ 重点：`UNITREE_NETWORK_IFACE`** 是最容易出错的参数！如果客户的 B2 不在 `192.168.123.x` 网段，或者你的 OrangePi 网卡名变了（比如插了 USB 网卡），这里必须改。

---

### Step 3：USB 摄像头推流

现场使用的是 USB 摄像头，需要启动推流脚本将其转为后端和前端都能使用的流。

```bash
# 插入 USB 摄像头后查看设备
ls /dev/video*

# 启动 USB 摄像头推流（假设为 /dev/video1，请根据实际情况修改）
./scripts/start_usb_cam.sh /dev/video1

# 此时 AI_RTSP_URL 必须为本机
# AI_RTSP_URL=rtsp://127.0.0.1:8554/cam
```

---

### Step 4：MediaMTX 视频中转配置

文件：`config/mediamtx.yml`

需要确认 `webrtcAdditionalHosts` 包含了 OrangePi 的实际 IP：

```yaml
webrtcAdditionalHosts:
  - 127.0.0.1
  - 192.168.123.222     # B2 网段的 IP
```

> **说明：** 如果笔记本通过 WiFi 访问 OrangePi，OrangePi 的 WiFi IP（如 192.168.3.x 等）也要加进这个列表，否则 WebRTC 视频画面出不来。

---

### Step 5：前端环境变量

文件：`frontend/.env`

如果前端是从 OrangePi 上 serve 的（静态文件由后端托管），不需要改。

如果前端是从笔记本本地 `npm run dev` 启动的，需要改为：

```ini
VITE_API_BASE_URL=http://<OrangePi实际IP>:8000
VITE_WHEP_URL=http://<OrangePi实际IP>:8889/cam/whep
VITE_STREAM_WIDTH=1280
```

---

### Step 6：自动跟踪参数（可现场微调）

当前 `.env` 里的跟踪参数已经过调试，但在客户 B2 上可能需要微调：

| 参数 | 当前值 | 说明 | 现场调整建议 |
|------|--------|------|-------------|
| `AUTO_TRACK_YAW_DEADBAND_PX` | 150 | 转向死区（px） | 画面抖动严重可加大到 200 |
| `AUTO_TRACK_FORWARD_AREA_RATIO` | 0.35 | 停止前进的面积阈值 | 室外可减到 0.25 让它走更近 |
| `AUTO_TRACK_YAW_PULSE_MS` | 150 | 转向脉冲时长 | B2 反应慢可加到 200 |
| `AUTO_TRACK_LOST_TIMEOUT_FRAMES` | 2 | 丢失超时帧数 | 室外光线变化大可加到 5 |
| `AI_CONFIDENCE_THRESHOLD` | 0.35 | 检测置信度 | 误检多可提到 0.5 |

> **提示：** 这些参数大部分也可以通过前端的**配置面板**实时修改，不用重启后端。

## 四、现场启动顺序

### 1. 物理连线

- B2 网线 → 交换机
- OrangePi 网线 → 交换机
- USB 摄像头 → 插入 OrangePi 的 USB 口

### 2. 配网（在 OrangePi 上）

```bash
sudo ip addr replace 192.168.123.222/24 dev enP3p49s0
sudo ip route replace 224.0.0.0/4 dev enP3p49s0
ping 192.168.123.161  # 验证 B2
```

### 3. 启动视频链路

```bash
cd ~/Code/Project/BOTDOG/BotDog
# 确保启动的是 USB 摄像头推流脚本，且传入正确的 video 设备号
bash scripts/start_usb_cam.sh /dev/video1
```

### 4. 启动后端

```bash
cd ~/Code/Project/BOTDOG/BotDog
source .venv/bin/activate
bash scripts/start_backend.sh
```

或者手动启动（需要先导出环境变量）：

```bash
export CYCLONEDDS_HOME=$HOME/cyclonedds/install
export LD_LIBRARY_PATH=$CYCLONEDDS_HOME/lib:${LD_LIBRARY_PATH:-}
PYTHONPATH=. python run_backend.py
```

### 5. 验证适配器

```bash
curl http://127.0.0.1:8000/api/v1/control/debug
```

确认输出中包含：
- `type = UnitreeB2Adapter`
- `initialized = true`
- `sport_client_exists = true`
- `worker_thread_alive = true`

### 6. 打开前端

笔记本浏览器访问：

```
http://<OrangePi_IP>:8000
```

确认：
- 遥测数据正常刷新
- 视频画面正常显示
- 控制按钮响应正常

### 7. 安全测试（必须按顺序！）

```
stand → stop → 低速 left/right → 低速 forward
```

确认运动正常后再演示自动跟踪功能。

## 五、应急备用方案

| 问题 | 快速解决 |
|------|---------|
| B2 ping 不通 | 确认 B2 开机、网线插好、IP 网段正确 |
| SDK 初始化失败 | 检查 `CYCLONEDDS_HOME` 环境变量、网卡名 |
| B2 不动但日志有命令 | B2 可能处于趴下/锁定状态，先用遥控器让它站起来 |
| 视频画面黑屏 | 先 VLC 验证 RTSP、检查 `mediamtx.yml` 的 `webrtcAdditionalHosts` |
| AI 不识别人 | 检查 `AI_ENABLED=true`、模型文件存在、RTSP 流正常 |
| 前端连不上后端 | 检查 OrangePi 防火墙 `sudo ufw status`、API URL 是否正确 |
| 所有功能都联不通 | `.env` 改 `CONTROL_ADAPTER_TYPE=simulation`、`AI_SIMULATE_DETECTION=true`，切模拟模式先演 |

---

## 六、特别注意事项

1. **客户的 B2 可能与你之前调试的 B2 不同**：网卡名、IP、固件版本都可能有差异
2. **首次在新 B2 上操作，务必低速**：先 `UNITREE_B2_VX=0.2`、`UNITREE_B2_VYAW=0.3`
3. **确保有人拿着遥控器做紧急制动**：遥控器优先级 > FT24 > Web 控制 > 自动跟踪
4. **CycloneDDS 环境变量**：`start_backend.sh` 会自动设置，但如果手动 `python run_backend.py` 启动，记得先 export
5. **演示自动跟踪前**：确认手动控制已经正常，再开启跟踪功能
6. **演示结束**：先停止跟踪 → stop → sit → 关闭后端 → 断开网线
