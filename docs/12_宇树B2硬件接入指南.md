# 宇树 B2 硬件接入指南

> **更新日期**：2026-04-08 | 适用平台：OrangePi 5 Ultra (ARM64)

---

## 一、整体部署架构

```
摄像头 (192.168.144.25, H.265 RTSP)
    │
HM30 图传网络 (192.168.144.x)
    │
OrangePi 5 Ultra（双网卡）
    ├── enP3p49s0: 192.168.123.222  ← 连接 B2 机器狗
    ├── enx...:    192.168.144.104  ← 连接 HM30/摄像头
    │
    ├── 后端 (FastAPI :8000)
    │     ├── UnitreeB2Adapter → SDK → B2
    │     └── AI Worker (YOLOv8) ← RTSP
    │
    ├── 视频流水线
    │     └── FFmpeg → MediaMTX → WHEP (:8889)
    │
地面端浏览器 (通过 HM30 访问 http://192.168.144.104:8000)
```

## 二、网络配置

### OrangePi 双网段配置

| 网卡 | IP | 用途 |
|------|----|------|
| `enP3p49s0` | `192.168.123.222/24` | 连接 B2（固定 IP） |
| `enx000000000426` | `192.168.144.104/24` | HM30 图传桥/摄像头（DHCP） |
| `wlan0` | 动态 | 可选 WiFi（用于 git pull 等外网操作） |

```bash
# 配置 B2 网段 IP
sudo ip addr replace 192.168.123.222/24 dev enP3p49s0

# 添加 DDS multicast 路由（Unitree SDK 必须！）
sudo ip route replace 224.0.0.0/4 dev enP3p49s0

# 验证连通性
ping 192.168.123.161   # B2（若 B2 IP 不同请改）
ping 192.168.144.25    # 摄像头
```

### 多网卡路由冲突修复（外网访问）

当 OrangePi 连了三张网卡但默认路由错乱时，可手动指定外网走 WiFi：
```bash
sudo ip route del default
sudo ip route add default via 192.168.111.1 dev wlan0
ping -c 3 github.com  # 验证
```

## 四、后端配置（`backend/.env`）

关键参数（真机运行时必须确认）：

```ini
# 控制器 → 宇树 B2
CONTROL_ADAPTER_TYPE=unitree_b2
UNITREE_NETWORK_IFACE=enP3p49s0   # OrangePi 连 B2 的网卡名（ip link show 确认）
UNITREE_B2_VX=0.3                  # 前进/后退速度 (m/s)，首次建议 0.2
UNITREE_B2_VYAW=0.5                # 转向速度 (rad/s)，首次建议 0.3

# Watchdog：B2 SDK RPC 有延迟，建议 1500ms
CONTROL_WATCHDOG_TIMEOUT_MS=1500

# 视频 AI
AI_ENABLED=true
AI_RTSP_URL=rtsp://127.0.0.1:8554/cam   # MediaMTX 转码后的本地流
AI_DEVICE=cpu
AI_MODEL_PATH=models/yolov8n.pt

# 服务监听
BACKEND_HOST=0.0.0.0
BACKEND_PORT=8000
```

## 五、视频流水线

摄像头输出 H.265，需要经过 FFmpeg 软解转码后才能在浏览器 WebRTC 播放：

```
IP 摄像头 (192.168.144.25 H.265 RTSP)
    ↓  FFmpeg 软解 + libx264 重编码（15fps, 800kbps, -bf 0, -fflags nobuffer）
MediaMTX RTSP (:8554/cam)
    ↓  WebRTC WHEP 协议
浏览器 <video> 播放
```

启动命令：
```bash
bash scripts/run-pipeline.sh
```

> **延迟说明**：由于 IP 摄像头硬件出流 H.265 无法修改，当前软解转码延迟约 200-400ms，为软件层面极限。
> 如需进一步降低，需替换为原生输出 H.264 的摄像头。

## 六、控制方式

### 6.1 Web 前端键盘遥控

| 按键 | 命令 | 说明 |
|------|------|------|
| `W` | `forward` | 前进 |
| `S` | `backward` | 后退 |
| `A` | `strafe_left` | 左平移 |
| `D` | `strafe_right` | 右平移 |
| `Q` | `left` | 左转 |
| `E` | `right` | 右转 |
| `Shift` | `stand` | 起立（RecoveryStand） |
| `Ctrl` | `sit` | 下蹲（StandDown） |

> 键盘遥控在任意页面内生效（输入框聚焦时自动屏蔽）。

### 6.2 控制优先级

```
E-STOP  >  遥控器（FT24）  >  Web 手动  >  AutoTrack 自动跟踪
```

### 6.3 API 调试

```bash
curl -X POST http://127.0.0.1:8000/api/v1/control/command \
     -H "Content-Type: application/json" \
     -d '{"cmd": "forward"}'
```

| 项目命令 | SDK 调用 | 说明 |
|---------|---------|------|

## 九、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `[UnitreeB2] 初始化失败` | SDK 未安装或网卡名错误 | 检查安装 + `ip link show` 确认网卡名 |
| ping 不通 B2 | IP 不在同网段 | 设置 `enP3p49s0` 为 `192.168.123.222` |
| B2 不动但有日志 | 机器人处于趴下/锁定状态 | 先用遥控器或 Shift 键让它站起来 |
| 视频 WHEP 404 | FFmpeg 推流未启动或摄像头离线 | 检查 `logs/ffmpeg.log`，重启 `run-pipeline.sh` |
| git pull 失败（Network unreachable） | 多网卡路由冲突 | `sudo ip route add default via 192.168.111.1 dev wlan0` |
| 前端显示"未连接" | 后端未运行或 API 地址错误 | 确认后端已启动；前端通过 `window.location.origin` 自动寻址 |
