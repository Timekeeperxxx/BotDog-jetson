# BotDog 智能巡检与自动取证系统功能架构设计

## 1. 概述

本设计旨在将 BotDog 从一台“带摄像头的遥控机器人”升级为具备自主感知能力的“智能巡检终端”。系统通过在后台并行分析视频流，实现对特定目标（如陌生人、违规垃圾等）的自动识别、取证与实时告警，同时确保不影响前端操控视频的低延迟体验。

## 2. 核心功能模块

### 2.1 旁路智能观察模块 (Side-Stream Observation)

该模块负责在后台“静默”观察现场状况，是系统的眼睛。

* ​**非侵入式扫描**​：在不干扰主控视频流（WHEP）的前提下，通过旁路获取视频数据。
* ​**低功耗运行模式**​：支持按需调节采样频率。在常规巡逻时使用低帧率扫描以节省电力和算力；在发现可疑目标时自动提升采样率。
* ​**目标多类识别**​：支持同时识别多种目标类别，包括但不限于人员（Person）、特定废弃物（如瓶罐、纸盒）以及用户定义的特定障碍物。

### 2.2 智能报警判定机制 (Multi-stage Decision)

该模块负责过滤环境干扰，确保每一条告警都是“深思熟虑”的结果。

* ​**时域稳定性校验**​：通过连续帧检测算法，确保目标在画面中稳定出现（而非闪现或阴影误报）后才触发告警。
* ​**智能告警消抖（冷却）**​：针对同一目标或同一区域的重复出现，实施分钟级的告警冷却逻辑，防止证据库被重复画面淹没。
* ​**优先级分级**​：根据检测目标的不同（如“陌生人”优先级高于“垃圾”），触发不同级别的处理流程。

### 2.3 自动瞬时取证模块 (Automatic Evidence Capture)

当判定逻辑确认异常后，系统自动执行“取证”动作。

* ​**高清瞬时抓拍**​：在检测到目标的瞬间，从原始数据流中提取并保存一张高分辨率图像。
* ​**多维数据关联**​：
  * ​**空间信息**​：自动关联抓拍瞬间的 GPS 经纬度。
  * ​**时间信息**​：记录精确到毫秒的系统时间戳。
  * ​**状态信息**​：记录当时的机器狗姿态（高度、角度）及任务 ID。
* ​**证据本地持久化**​：将图片与结构化元数据安全存储在本地文件系统与数据库中。

### 2.4 实时异步推送模块 (Real-time Notification)

负责将发现的异常第一时间反馈给后方操作员。

* ​**即时 WebSocket 广播**​：利用现有通信链路，在 500ms 内将告警信号推送到所有控制终端。
* ​**卡片式预览**​：通知消息需包含抓拍缩略图、目标类型及可信度分数，让操作员无需切换界面即可完成初步研判。
* ​**声光提示支持**​：支持触发前端警报音或机器人端灯光反馈。

### 2.5 巡检档案管理模块 (Historical Traceability)

负责对所有的巡检取证结果进行数字化管理。

* ​**异常证据链回溯**​：提供完整的时间线视图，用户可以按时间、地点或任务名称筛选历史抓拍。
* ​**地理坐标映射**​：在地图插件中自动标注异常发现点，支持点击图标跳转查看对应的现场图片。
* ​**数据自动清理**​：根据磁盘剩余空间和保存期限，自动执行证据图片的循环覆盖，确保存储系统不宕机。

### 2.6 系统自保护与环境联动 (System Resilience)

确保 AI 功能不会拖垮机器狗的基础运动能力。

* ​**温控动态降级**​：与硬件温度监控联动。当主控温度过高时，系统自动降低 AI 检测频率或暂时挂起分析任务。
* ​**状态驱动激活**​：AI 功能仅在机器狗处于“任务执行”状态时激活，待机或手动模式下自动转入低能耗休眠。

## 3. 系统业务流向

1. ​**观察阶段**​：后台 Worker 持续从视频枢纽获取画面字节。
2. ​**分析阶段**​：AI 逻辑进行目标检测 -> 连续帧确认 -> 过滤冷却。
3. ​**取证阶段**​：触发抓拍 -> 调用数据库服务存储证据 -> 关联 GPS。
4. ​**反馈阶段**​：通过广播器推送 WebSocke 消息 -> 前端 UI 弹出实时告警。
5. ​**归档阶段**​：操作员在证据历史记录中查看、标记或清理取证文件。

## 4. 架构设计的核心价值

* ​**操控不掉帧**​：AI 逻辑完全旁路化，不占用主视频解码链路。
* ​**证据可追溯**​：每一张图都有坐标和时间，具备极高的巡检参考价值。
* ​**自适应可靠​**​：在硬件高温或资源紧张时能自动退避，保证机器狗“不死机、不失控”。
