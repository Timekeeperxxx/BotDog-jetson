BotDog 前端控制架构设计
本架构设计针对前端网页控制机器狗的需求，采用 HTTP 请求进行动作控制，后端服务通过 Control Service 进行动作判断和安全性保证。

1. 前端控制面板：
- 控制按钮包括前进、后退、左、右、下蹲、起立。
- 按下按钮时，前端持续发 HTTP 请求；松手时，立即发 stop 请求。

2. 控制服务层（Control Service）：
- 判断当前系统是否允许控制
- 调用设备适配器转发控制命令
- 对动作进行超时监控，确保及时停止动作。

3. 设备适配器（Robot Adapter）：
- 对接当前测试狗的控制接口，未来可以替换为目标机器狗的适配器。

4. 安全性设计：
- 前端失焦、离开控制区、页面卸载等事件自动触发 stop。
- 后端增加 watchdog 机制，超过超时未收到命令自动执行 stop。

5. 演进路径：
- 阶段性目标从简单的控制命令发放，到可扩展的控制域管理，最后支持多控制源与设备适配。
