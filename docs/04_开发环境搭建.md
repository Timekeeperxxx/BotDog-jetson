# 开发环境搭建指南

> **适用平台**：OrangePi 5 Ultra (ARM64 / Ubuntu 22.04)  
> **更新日期**：2026-04-08

---

## 一、硬件需求

| 设备 | 规格 | 备注 |
|------|------|------|
| OrangePi 5 Ultra | RK3588 / 8GB+ RAM | 主控板，安装在机器狗背上 |
| Unitree B2 | — | 四足机器人本体 |
| IP 摄像头 | SIYI 系列，H.265 RTSP | `192.168.144.25:8554/main.264` |
| HM30 图传 | 天空端+地面端 | 无线图传+遥控链路 |
| 千兆交换机 | — | 连接摄像头、OrangePi、HM30 |
| 地面端电脑 | 现代浏览器 | Chrome/Edge 推荐 |

## 二、OrangePi 系统环境

### 2.1 Python 虚拟环境

```bash
cd ~/Code/Project/BOTDOG/BotDog

# 创建虚拟环境
python3 -m venv .venv
source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### 2.2 Unitree SDK 安装

```bash
# CycloneDDS（DDS 中间件，Unitree SDK 依赖）
pip install cyclonedds==0.10.2

# Unitree SDK2 Python binding
git clone https://github.com/unitreerobotics/unitree_sdk2_python.git
cd unitree_sdk2_python && pip install -e . && cd ..
```

### 2.3 YOLO 模型文件

```bash
mkdir -p models
# 下载 YOLOv8n 模型（约 6MB）
# https://github.com/ultralytics/assets/releases/download/v8.2.0/yolov8n.pt
# 放入 models/ 目录
```

### 2.4 MediaMTX

```bash
# 下载 ARM64 版本 MediaMTX
cd scripts/
wget https://github.com/bluenviron/mediamtx/releases/download/v1.9.3/mediamtx_v1.9.3_linux_arm64.tar.gz
tar -xzf mediamtx_v1.9.3_linux_arm64.tar.gz mediamtx
chmod +x mediamtx
```

## 三、前端构建

```bash
cd frontend
npm install
npm run build   # 构建生产版本，输出到 frontend/dist/
```

构建后后端会自动 serve `frontend/dist/` 的静态文件，浏览器直接访问 `http://<OrangePi_IP>:8000` 即可。

> **如需前端开发模式**（热重载）：
> ```bash
> npm run dev -- --host 0.0.0.0
> # 访问 http://<OrangePi_IP>:5174
> ```

---

## 四、后端配置

```bash
cp backend/.env.example backend/.env
nano backend/.env   # 按需修改
```

**关键配置项**：

```ini
# 控制适配器（真机）
CONTROL_ADAPTER_TYPE=unitree_b2
UNITREE_NETWORK_IFACE=enP3p49s0

# 模拟模式（无机器狗开发时）
CONTROL_ADAPTER_TYPE=simulation
SIMULATION_WORKER_ENABLED=true
AI_SIMULATE_DETECTION=true

# AI
AI_ENABLED=true
AI_RTSP_URL=rtsp://127.0.0.1:8554/cam
AI_MODEL_PATH=models/yolov8n.pt
AI_DEVICE=cpu
```

## 五、启动顺序

```bash
cd ~/Code/Project/BOTDOG/BotDog
source .venv/bin/activate

# Step 1：初始化数据库（首次运行）
python scripts/init_db.py

# Step 2：启动视频流水线
bash scripts/run-pipeline.sh

# Step 3：启动后端
bash scripts/start_backend.sh

# Step 4：浏览器访问
# http://<OrangePi_IP>:8000
```

## 六、开机自启（systemd 服务）

```bash
# 安装 systemd 服务（需 sudo）
sudo bash scripts/install-services.sh

# 手动管理
sudo systemctl start botdog-backend
sudo systemctl start botdog-pipeline
sudo systemctl status botdog-backend

# 查看实时日志
journalctl -u botdog-backend -f
journalctl -u botdog-pipeline -f
```

## 七、快捷命令参考

```bash
# 后端开发模式（热重载）
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000

# 查看视频流日志
tail -f logs/ffmpeg.log
tail -f logs/mediamtx.log

# 停止视频流水线
bash scripts/run-pipeline.sh stop

# 验证 B2 适配器状态
curl http://127.0.0.1:8000/api/v1/control/debug

# 数据库重置
rm -f data/botdog.db && python scripts/init_db.py
```

## 八、纯模拟模式（无硬件开发）

适合在 Windows/Mac 上快速开发前端或调试后端逻辑：

```bash
# backend/.env
CONTROL_ADAPTER_TYPE=simulation
SIMULATION_WORKER_ENABLED=true
AI_ENABLED=false

# 前端开发
cd frontend && npm run dev
# 后端
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```
