# BotDog 视频流系统启动指南

## 系统架构

```
相机 (RTSP) → UDP 中继 → 硬件解码 → WebRTC → 浏览器
192.168.144.25    5000       D3D11      HTTP    8000
```

## 网络配置

- **相机 IP**: `192.168.144.25:8554` (RTSP)
- **系统 IP**: `192.168.144.30`
- **UDP 端口**: `5000` (视频接收)
- **Web 端口**: `8000` (FastAPI)

## 启动步骤

### 1. 启动后端服务

```bash
cd D:\ACode\Project\BOTDOG\BotDog
python run_backend.py
```

**预期输出**:
```
INFO:     Started server process [xxxxx]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000
```

**如果看到警告** (正常情况):
```
[警告] UDP 端口 5000 暂时不可用，等待视频流...
这通常是因为相机还未开始推流
```

### 2. 启动相机推流

**新开一个终端窗口**:

```bash
cd D:\ACode\Project\BOTDOG\BotDog
python camera_streamer.py
```

**预期输出**:
```
================================================================================
相机 RTSP 推流器
================================================================================

相机: rtsp://192.168.144.25:8554/main.264
目标: 192.168.144.30:5000

[方案 1] 使用 GStreamer
管道: gst-launch-1.0 -q rtspsrc location=rtsp://192.168.144.25:8554/main.264 ...

启动推流...
按 Ctrl+C 停止

[OK] 推流进程已启动 (PID: xxxxx)
```

### 3. 验证视频流

#### 方法 1: 检查 GPU 使用率

打开 **任务管理器** → **性能** → **GPU** → **Video Decoding**

如果看到 GPU 有负载，说明硬件解码已激活 ✅

#### 方法 2: 浏览器查看

打开浏览器访问:
```
http://localhost:8000
```

在 WebRTC 控制台应该能看到视频流。

### 4. 停止系统

按 `Ctrl+C` 停止相机推流
按 `Ctrl+C` 停止后端服务

## 故障排查

### 问题 1: 后端启动失败

**错误**: `ImportError: attempted relative import`

**解决**: 确保使用 `run_backend.py` 而不是直接运行 `main.py`

### 问题 2: UDP 端口绑定失败

**错误**: `[WinError 10049] 在其上下文中，该请求的地址无效`

**状态**: 已修复 ✅
系统会显示警告但继续启动，等待相机推流后会自动连接

### 问题 3: 相机无法连接

**检查**:
1. 相机是否开机: `ping 192.168.144.25`
2. RTSP 是否可达: `ffplay rtsp://192.168.144.25:8554/main.264`

### 问题 4: 视频显示灰屏

**原因**: 软件解码性能不足

**解决**: 已配置硬件解码器 `d3d11h265dec`，检查 GPU 是否被正确使用

## 技术规格

### 硬件解码器
- **解码器**: `d3d11h265dec` (Direct3D 11 H.265)
- **GPU**: NVIDIA RTX 3060
- **性能**: ~37 FPS (已验证)

### 视频参数
- **编码**: H.265 (HEVC)
- **分辨率**: 1920x1080 (1080p)
- **帧率**: 30 FPS
- **传输**: RTP over UDP

### 软件版本
- **Python**: 3.10.x
- **GStreamer**: 1.28.1 (MSVC x86_64)
- **操作系统**: Windows 11

## 配置文件

所有配置位于 [backend/config.py](backend/config.py):

```python
CAMERA_RTSP_URL: str = "rtsp://192.168.144.25:8554/main.264"
UDP_RELAY_LISTEN_PORT: int = 5000
UDP_RELAY_BIND_ADDRESS: str = "192.168.144.30"
VIDEO_RESOLUTION: str = "1920x1080"
VIDEO_BITRATE: int = 8000000  # 8 Mbps
```

## 性能优化建议

1. **网络**: 使用有线连接，避免 WiFi 干扰
2. **GPU**: 确保 GPU 驱动已更新到最新版本
3. **码率**: 根据网络状况调整 `VIDEO_BITRATE`
4. **延迟**: 降低 `rtpjitterbuffer latency` 以减少延迟（可能增加丢帧）
