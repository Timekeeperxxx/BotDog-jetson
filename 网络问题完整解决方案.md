# BotDog 视频流系统 - 网络问题完整解决方案

## 问题诊断

### 根本原因
**路由冲突**: WiFi 和以太网同时启用时，Windows 错误地使用 WiFi 接口转发机器人流量，导致无法连接到相机 IP (192.168.144.25:8554)。

### 症状
- ✅ Ping 相机成功
- ❌ RTSP 连接超时
- ❌ GStreamer 错误: "Timeout while waiting for server response"
- 📊 网络跟踪显示流量源地址为 WiFi 网段 (192.168.18.x) 而非以太网网段 (192.168.144.30)

## 解决方案

### 方案 1: 关闭 WiFi（推荐，最简单）

1. 打开 Windows 设置
2. 进入 **网络和 Internet** → **Wi-Fi**
3. 点击 **关闭 Wi-Fi**

**优点**: 立即生效，无需重启
**缺点**: 失去 WiFi 互联网连接

---

### 方案 2: 调整网卡优先级（永久方案）

#### 自动配置（推荐）

1. 以管理员身份运行 PowerShell
2. 执行脚本:
   ```powershell
   cd D:\ACode\Project\BOTDOG\BotDog
   .\fix_network_priority.ps1
   ```
3. 重启计算机

#### 手动配置

1. 打开 **控制面板** → **网络和共享中心**
2. 点击 **更改适配器设置**
3. 右键 **以太网** → **属性**
4. 双击 **Internet 协议版本 4 (TCP/IPv4)**
5. 点击 **高级**
6. 取消勾选 **自动跃点**
7. 设置 **跃点数** 为 **1**
8. 对 WiFi 适配器重复上述步骤，设置跃点数为 **9999**
9. 重启计算机

**优点**: WiFi 和以太网可同时使用
**缺点**: 需要重启计算机

---

### 2. 指数退避重连机制

**位置**: `backend/video_track_reconnecting.py`

**功能**:
- 自动重连机制
- 指数退避（Exponential Backoff）
- 最大重试次数限制
- 网络恢复后自动恢复视频流

**参数**:
```python
max_retries=10          # 最大重试次数（0=无限重试）
initial_delay=1.0       # 初始重试延迟（秒）
max_delay=60.0          # 最大重试延迟（秒）
backoff_multiplier=2.0  # 退避乘数
```

**重连时间表**:
| 尝试次数 | 延迟时间 |
|---------|---------|
| 1       | 1秒     |
| 2       | 2秒     |
| 3       | 4秒     |
| 4       | 8秒     |
| 5       | 16秒    |
| 6       | 32秒    |
| 7+      | 60秒    |

**使用**:
在 `webrtc_signaling.py` 中替换：
```python
# 原代码
from .video_track_native import GStreamerVideoSourceFactory

# 新代码
from .video_track_reconnecting import ReconnectingVideoTrack
```

## 故障排查

### 问题 1: 网络检查失败

**症状**:
```
[警告] 检测到 WiFi 和以太网同时启用
```

**解决方案**:
1. 关闭 WiFi
2. 或运行 `fix_network_priority.ps1`

---

### 问题 2: RTSP 连接超时

**症状**:
```
ERROR: Timeout while waiting for server response
```

**解决方案**:
1. 检查相机是否开机
2. 验证网络路由（运行网络检查）
3. 使用测试源验证系统（`python push_test_stream.py`）

---

### 问题 3: 视频流灰屏

**症状**: 浏览器显示灰色画面

**解决方案**:
1. 检查 GPU 硬件解码是否启用（任务管理器）
2. 验证 GStreamer 管道是否正常启动
3. 查看后端日志中的错误信息

---

### 问题 4: WebRTC 连接失败

**症状**:
```
WebRTC信令连接已关闭: code=1006
```

**解决方案**:
1. 重启后端服务
2. 清除浏览器缓存
3. 检查防火墙设置
